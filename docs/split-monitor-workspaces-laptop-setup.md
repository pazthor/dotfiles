## Overview

This setup provides **dedicated workspace ranges per monitor** using native Hyprland workspace rules. Each monitor has its own set of workspaces that are explicitly assigned and cannot move to other monitors.

## Problem Statement

By default, Hyprland uses global workspace numbering (1-10). In a dual-monitor setup, this means:
- Workspaces can float between monitors
- You need to remember which workspace numbers belong to which monitor
- Duplicate workspace IDs can appear on both monitors

**Desired behavior**: Each monitor should have dedicated workspaces that stay locked to that monitor.

## Solution: Native Hyprland Workspace Rules

Hyprland's built-in `workspace` rules allow explicit per-monitor workspace assignments:

```conf
workspace=ID,monitor:NAME,default:BOOL
```

This approach is:
- ✅ Simple and reliable
- ✅ No plugin dependencies
- ✅ No version compatibility issues
- ✅ Survives Hyprland updates
- ⚠️ Shows actual workspace numbers (1-5 on laptop, 6-10 on external)

## Configuration

### 1. Monitor and Workspace Configuration

The configuration is managed in hyprdynamicmonitors templates for dynamic monitor switching.

**File**: `.config/hyprdynamicmonitors/hyprconfigs/dual-external-top.go.tmpl`

```go
{{- $laptop := index .MonitorsByTag "laptop" -}}
{{- $external := index .MonitorsByTag "external" }}

# Autogenerated by hyprdynamicmonitors - External monitor on top, laptop below
# External monitor at top (0x0) - AOC 34" ultrawide at preferred resolution
monitor={{$external.Name}},preferred,0x0,1.67,vrr,{{.vrr_on_ac}}

# Laptop monitor below external (using logical pixels with scaling)
# X: (3440/1.67 - 1920/1.5) / 2 ≈ (2060 - 1280) / 2 = 390 for centering
# Y: 1440/1.67 ≈ 862.87 rounds to 863, so laptop needs to start at 864 to avoid overlap
monitor={{$laptop.Name}},{{.default_res}}@{{.ac_refresh}},390x864,{{.ac_scale}},vrr,{{.vrr_on_ac}}

# Disable any unexpected extra monitors
{{- range .ExtraMonitors}}
monitor={{.Name}},disable
{{- end }}

# Workspace assignments - Per-monitor workspace ranges
# Laptop (eDP-1): workspaces 1-5
workspace=1,monitor:{{$laptop.Name}},default:true
workspace=2,monitor:{{$laptop.Name}}
workspace=3,monitor:{{$laptop.Name}}
workspace=4,monitor:{{$laptop.Name}}
workspace=5,monitor:{{$laptop.Name}}

# External (HDMI-A-1): workspaces 6-10
workspace=6,monitor:{{$external.Name}},default:true
workspace=7,monitor:{{$external.Name}}
workspace=8,monitor:{{$external.Name}}
workspace=9,monitor:{{$external.Name}}
workspace=10,monitor:{{$external.Name}}

# Enable cursor-based monitor/window focus
input {
    follow_mouse = 1  # 1=always follow, 2=floating only, 3=no focus stealing
}
```

**Key Configuration Points:**

1. **Monitor Positioning**:
   - External at (0, 0)
   - Laptop centered below at (390, 864)
   - **Critical**: Y=864 prevents monitor overlap warnings (not 862 or 863!)

2. **Workspace Assignments**:
   - `default:true` on workspace 1 and 6 makes them the initial workspaces
   - Each workspace is locked to its monitor and cannot move

### 3. Keybinding Configuration

Standard Hyprland workspace dispatchers with explicit workspace numbers.

**File**: `.config/hypr/bindings.conf`

```conf
# Workspace switching - SUPER + number
bindd = SUPER, 1, Workspace 1, workspace, 1
bindd = SUPER, 2, Workspace 2, workspace, 2
bindd = SUPER, 3, Workspace 3, workspace, 3
bindd = SUPER, 4, Workspace 4, workspace, 4
bindd = SUPER, 5, Workspace 5, workspace, 5
bindd = SUPER, 6, Workspace 6, workspace, 6
bindd = SUPER, 7, Workspace 7, workspace, 7
bindd = SUPER, 8, Workspace 8, workspace, 8
bindd = SUPER, 9, Workspace 9, workspace, 9
bindd = SUPER, 0, Workspace 10, workspace, 10

# Move active window to workspace - SUPER + SHIFT + number
bindd = SUPER SHIFT, 1, Move to workspace 1, movetoworkspacesilent, 1
bindd = SUPER SHIFT, 2, Move to workspace 2, movetoworkspacesilent, 2
bindd = SUPER SHIFT, 3, Move to workspace 3, movetoworkspacesilent, 3
bindd = SUPER SHIFT, 4, Move to workspace 4, movetoworkspacesilent, 4
bindd = SUPER SHIFT, 5, Move to workspace 5, movetoworkspacesilent, 5
bindd = SUPER SHIFT, 6, Move to workspace 6, movetoworkspacesilent, 6
bindd = SUPER SHIFT, 7, Move to workspace 7, movetoworkspacesilent, 7
bindd = SUPER SHIFT, 8, Move to workspace 8, movetoworkspacesilent, 8
bindd = SUPER SHIFT, 9, Move to workspace 9, movetoworkspacesilent, 9
bindd = SUPER SHIFT, 0, Move to workspace 10, movetoworkspacesilent, 10

# Monitor operations - move current workspace to another monitor
bindd = SUPER CTRL, H, Move workspace to left monitor, movecurrentworkspacetomonitor, l
bindd = SUPER CTRL, L, Move workspace to right monitor, movecurrentworkspacetomonitor, r

# Move window to adjacent monitor
bindd = SUPER CTRL SHIFT, H, Move window to left monitor, movewindow, mon:l
bindd = SUPER CTRL SHIFT, L, Move window to right monitor, movewindow, mon:r
```

**Key Dispatchers:**
- `workspace N` - Switch to workspace N (will only work if on correct monitor)
- `movetoworkspacesilent N` - Move window to workspace N without following
- `movecurrentworkspacetomonitor` - Move entire workspace to different monitor
- `movewindow mon:l/r` - Move window to adjacent monitor

## How It Works

### Workspace Assignment

The `workspace` rules explicitly assign workspaces to monitors:

- **Monitor eDP-1 (laptop)**: Workspaces 1-5
- **Monitor HDMI-A-1 (external)**: Workspaces 6-10

Once assigned, workspaces cannot spontaneously move to other monitors.

### Workspace Switching Behavior

- On laptop: `SUPER+1-5` switches between laptop workspaces
- On external: `SUPER+6-9,0` switches between external workspaces
- Pressing `SUPER+6` while on laptop switches focus to external monitor
- Workspaces are globally numbered (no remapping like the plugin would have done)

### Monitor Positioning Math

**Why Y=864?**

1. External monitor: 3440x1440 @ 1.67 scale
2. Logical height: 1440 ÷ 1.67 = 862.87 pixels
3. Hyprland rounds 862.87 to 863
4. External occupies Y coordinates: 0-863 (inclusive)
5. Laptop must start at Y=864 to avoid overlap

**Common mistakes:**
- ❌ Y=862 - Overlaps by 1 pixel
- ❌ Y=863 - Boundary overlap warning
- ✅ Y=864 - Perfect spacing, no warnings

## Usage

### Basic Workspace Switching

**Laptop workspaces (1-5):**
- `SUPER+1-5` - Switch to workspace 1-5
- `SUPER+SHIFT+1-5` - Move window to workspace 1-5

**External workspaces (6-10):**
- `SUPER+6-9,0` - Switch to workspace 6-10
- `SUPER+SHIFT+6-9,0` - Move window to workspace 6-10

### Moving Between Monitors

**Move workspace:**
- `SUPER+CTRL+H/L` - Move entire current workspace to left/right monitor

**Move window:**
- `SUPER+CTRL+SHIFT+H/L` - Move active window to left/right monitor

### Behavior Examples

1. **Start on laptop workspace 1**, press `SUPER+6`:
   - Focus switches to external monitor workspace 6
   - Cursor may move to external monitor (depending on settings)

2. **Window on laptop workspace 2**, press `SUPER+SHIFT+7`:
   - Window moves to external monitor workspace 7
   - You stay on laptop workspace 2

3. **On external workspace 8**, press `SUPER+CTRL+H`:
   - Entire workspace 8 moves to laptop monitor
   - Workspace 8 is now on laptop (rules allow temporary overrides via dispatcher)

```bash

hyprctl dispatch moveworkspacetomonitor 3 eDP-1
hyprctl dispatch moveworkspacetomonitor 4 eDP-1
hyprctl dispatch moveworkspacetomonitor 5 eDP-1

# Move workspaces 6-10 to external (if needed)
hyprctl dispatch moveworkspacetomonitor 6 HDMI-A-1
hyprctl dispatch moveworkspacetomonitor 7 HDMI-A-1

hyprctl dispatch moveworkspacetomonitor 9 HDMI-A-1

```

New workspaces will automatically be created on the correct monitor per the rules.

## Troubleshooting

### Issue: Monitor Overlap Warning

**Symptom**: `hyprctl reload` shows "monitor HDMI-A-1 overlaps with other monitor"

**Cause**: Laptop Y-position is too low (862 or 863 instead of 864)

**Solution**: Ensure laptop Y-position is 864 in `dual-external-top.go.tmpl`:
```go
monitor={{$laptop.Name}},{{.default_res}}@{{.ac_refresh}},390x864,{{.ac_scale}},vrr,{{.vrr_on_ac}}
```

### Issue: Workspace on Wrong Monitor

**Symptom**: Workspace 1 appears on external monitor instead of laptop

**Cause**: Workspace was created before workspace rules were applied

**Solution**: Manually move the workspace:
```bash
hyprctl dispatch moveworkspacetomonitor 1 eDP-1
```

### Issue: Can't Switch to Workspace

**Symptom**: Pressing `SUPER+6` doesn't work when on laptop

**Cause**: This is expected! Workspace 6 is on external monitor

**Solution**: Either:
1. Move cursor to external monitor first, then press `SUPER+6`
2. Use `SUPER+CTRL+SHIFT+L` to move window directly to external monitor

## Integration with hyprdynamicmonitors

The workspace rules work seamlessly with hyprdynamicmonitors:

1. **Templates** contain both monitor and workspace configs
2. **Auto-regeneration** happens on power state or monitor connection changes
3. **Laptop-only mode** automatically uses only workspaces 1-5

When the external monitor is disconnected:
- Workspaces 6-10 become inaccessible (no monitor assigned)
- When monitor reconnects, workspaces 6-10 return to external monitor
- Windows on workspaces 6-10 remain on those workspaces (not moved to laptop)
